# `edgereplica-protocol`

Wire formats for EdgeReplica. Server (`edgereplica-worker`) and client
(`edgereplica-client`) both link against this crate, so they decode against
the same types.

Two protocols live side-by-side:

| Module             | Surface                                            | Codec                            |
| ------------------ | -------------------------------------------------- | -------------------------------- |
| `protocol::admin`  | `edgereplica.admin.v1.AdminService` (ConnectRPC)   | Protobuf or JSON over HTTP/HTTPS |
| `protocol::sync`   | `SyncMessage` frames over a single WebSocket       | MessagePack with version prefix  |

`protocol::admin` is generated by `connectrpc-build` from
`../proto/edgereplica/admin/v1/admin.proto` at build time (see
`build.rs`). The generated tree is wrapped in a private `mod generated`
inside `src/lib.rs` and re-exported as `protocol::admin`. `protocol::sync`
is hand-written in `src/sync.rs`.

---

## Admin RPC (`protocol::admin`)

Unary ConnectRPC over HTTP. Default codec is protobuf (`application/proto`);
the connectrpc client also negotiates JSON via the standard ConnectRPC
content-type rules.

### Methods

| Method                          | Auth required | Notes                                                            |
| ------------------------------- | ------------- | ---------------------------------------------------------------- |
| `Whoami`                        | session       | `idempotency_level = NO_SIDE_EFFECTS`                            |
| `Signup`                        | none          | Creates user + personal org + admin membership atomically (D1 batch) |
| `Login`                         | none          | Email + password; returns a fresh session token                  |
| `StartOAuth`                    | none          | Returns redirect URL + opaque state; gated by env config         |
| `CompleteOAuth`                 | none          | Exchanges IdP `code` for a session token                         |
| `CreateDatabase`                | session+admin | Name unique within org, ASCII alphanumerics + `-` `_`, ≤64 chars |
| `ListDatabases`                 | session       | `idempotency_level = NO_SIDE_EFFECTS`                            |
| `DeleteDatabase`                | session+admin | Org-scoped — refuses cross-org deletion                          |
| `IssueSyncToken`                | session       | Mints a sync macaroon for `(database, direction)`                |

Sessioned RPCs require an `Authorization: Bearer <session-macaroon>` header.
Bad/missing tokens on session-required RPCs return ConnectRPC code
`Unauthenticated`. Direction enum:
`DIRECTION_UNSPECIFIED | DIRECTION_PUSH | DIRECTION_PULL`.

### URLs

ConnectRPC's path scheme: `POST /<package>.<Service>/<Method>` — e.g.
`POST /edgereplica.admin.v1.AdminService/Whoami`. The worker also serves
two non-RPC routes outside this prefix: `GET /healthz` and the OAuth
callback at `GET /oauth/<provider>/callback`.

### Regenerating

`cargo build -p edgereplica-protocol` re-runs `build.rs` whenever
`admin.proto` changes. The TypeScript admin client used by
`integration-tests/` is regenerated separately via
`pnpm -C integration-tests run generate` (buf-es, output in
`integration-tests/gen/`).

## Sync wire format (`protocol::sync`)

One binary WebSocket carries the whole sync conversation. Text frames are
illegal and terminate the connection.

### Frame layout

```
+--------+----------------------------+
|  ver   |   msgpack(SyncMessage)     |
| 1 byte |  variable length           |
+--------+----------------------------+
```

`ver` is `PROTOCOL_VERSION` (currently `1`). Peers reject mismatched
frames before paying msgpack-decode cost. The body is `SyncMessage`
serialized with `rmp-serde::to_vec_named` (msgpack named-field map
encoding). The enum is adjacently tagged so a debugger or
`msgpack-cli | jq` shows
`{"kind":"page_hash","data":{"page_no":42,"hash":"<32 bytes>"}}`
instead of an opaque integer discriminant.

Page hashes are 32-byte BLAKE3 digests carried as `bytes::Bytes`, which
encodes to msgpack `bin` (not a sequence). BLAKE3 was chosen over SHA-256
because the worker runs in wasm with no SHA-256 hardware acceleration —
`bench/wasm-bench` measured BLAKE3 at ~5x SHA-256 throughput on wasm.

### Phases

Push and pull both use the same envelope; the FSM (in
`worker/src/services/sync_fsm.rs`) enforces who may send what based on the
direction baked into the macaroon. Direction is **not** on the wire.

1. **Handshake.** Client sends `Hello { protocol_version, page_size, max_page }`.
   Server replies `HelloReply` with its own `max_page`. A protocol-version
   mismatch surfaces here as a fatal `Error`.
2. **Hash phase.** Each peer sends `PageHash { page_no, hash }` (or batches
   via `PageHashBatch` for whole-range short-circuits) for every page it
   has. The receiver compares and, depending on direction, either issues
   `RequestPage`/`RequestPages` (push) or streams `PageData` inline (pull).
3. **Drain.** Client closes its hash phase with `Complete`. Push: server
   waits for any outstanding `RequestPage` replies, then pull-style
   reconciliation runs over pages the client never claimed. Pull: server
   walks pages the client didn't claim and ships each as `PageData`.
4. **Termination.** Server sends `SyncComplete { pages_transferred,
   bytes_transferred }` exactly once and closes with WebSocket code
   `1000`. Either peer can send `Error { message }` and close `1011` /
   `1003` / `1002` to abort.

### Message reference

| Variant              | Direction        | Body                                                   |
| -------------------- | ---------------- | ------------------------------------------------------ |
| `Hello`              | Client → Server  | `protocol_version`, `page_size`, `max_page`            |
| `HelloReply`         | Server → Client  | Same shape; `max_page` is server-side                  |
| `PageHash`           | Either           | `page_no`, 32-byte `hash`                              |
| `PageHashBatch`      | Either           | `start_page`, `end_page`, `combined_hash` (BLAKE3 over concat) |
| `RequestPage`        | Server → Client  | Single page (push)                                     |
| `RequestPages`       | Server → Client  | Batch (push)                                           |
| `HashMismatch`       | Server → Client  | Single page (pull hint)                                |
| `HashMismatchBatch`  | Server → Client  | Batch (pull hint)                                      |
| `PageData`           | Either           | `page_no`, `data` (raw page bytes)                     |
| `PageDataBatch`      | Either           | `Vec<PageDataEntry>`                                   |
| `Complete`           | Client → Server  | "I've sent every hash I have"                          |
| `SyncComplete`       | Server → Client  | `pages_transferred`, `bytes_transferred`               |
| `Error`              | Either           | `message`; fatal — connection closes                   |

The FSM ships each outbound frame via a callback that flushes immediately,
so a multi-GB pull never buffers the whole DB in worker RAM.

### Versioning

Bump `PROTOCOL_VERSION` (and matching `Hello.protocol_version`) when a
change in message semantics is something an older peer cannot parse —
e.g. removing or renaming a variant, changing the meaning of a field.
Pure additions don't need a bump. Older peers reject the new variant via
the FSM's unknown-variant path ("client may not send server-originated
message" or similar).

## Authentication

Both protocols use macaroons (`libmacaroon` Format V2) carried as
`Authorization: Bearer <token>`. Verification is pure (no DB read);
caveats are first-party only. Mint/verify code lives in
`worker/src/auth/tokens.rs`.

### Session token

```
location:  edgereplica
identifier: session:<user_id>
caveats:
  purpose=session
  user=<user_id>
  email=<email>
  org=<org_id>
  role=<admin|member>
  exp=<unix_seconds>
```

Issued by `Signup`, `Login`, `CompleteOAuth`. Default TTL 24h
(`SESSION_TTL_SECONDS`). Used on the admin RPC `Authorization` header.

### Sync token

```
location:  edgereplica
identifier: sync:<user_id>:<database_id>
caveats:
  purpose=sync
  user=<user_id>
  org=<org_id>
  database=<database_id>
  direction=<push|pull>
  exp=<unix_seconds>
```

Issued by `IssueSyncToken`. Default TTL 1h, capped at
`MAX_SYNC_TOKEN_TTL_SECONDS` (default 24h). Required on the WebSocket
upgrade request to `/sync`. Verified twice — once at the worker edge in
`sync_forward.rs`, once inside the DurableObject in `do_sync_ws.rs` —
because the DO never implicitly trusts the worker.

A session token presented as a sync token (or vice versa) is rejected:
either by libmacaroon's caveat verifier (unknown caveat for the wrong
purpose) or by the explicit `purpose=` check after verification, which
returns `TokenError::WrongPurpose`.

## Crate dependencies

`protocol` deliberately has no internal workspace dependencies. It pulls
in `connectrpc` + `buffa` for admin codegen, plus `rmp-serde`, `blake3`,
`bytes`, `serde`, `thiserror` for sync. All of these build on
`wasm32-unknown-unknown` so the worker can link the crate unchanged.
